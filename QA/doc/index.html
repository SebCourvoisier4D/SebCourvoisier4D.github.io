Wakanda QA Internal Tools

Purpose

The aim is to develop a single package of node.js modules that provides all the tools needed by the QA team to run the tests and report their results (either programmatically or using the command line).

qa-scripts/csjs-headless.js

(STABLE) Run the given test using PhantomJS/SlimerJS/TrifleJS. It means that this script has to be run using one of these headless tools, for instance:

phantomjs /path/to/qa-scripts/csjs-headless.js <test_URL> <report_path>

The script expects 2 arguments:

  1. test_URL: The URL of the test to run (e.g. http://127.0.0.1:8081/index.waPage/index.html?waktest-path=/full/path/to/the/test.js&waktest-format=JUnit)
  2. report_path: The path where the report will be created (as report.json, report.xml or report.html according to the given waktest-format URL parameter)

qa-scripts/rename-in-pom.js

(TO BE REMADE) Perform a search/replace in all the pom.xml and pom.json files:

node /path/to/qa-scripts/rename-in-pom.js

The search/replace patterns (as well as the P4 settings) have to be specified within the script itself (lines 18 to 26).

qa-scripts/studio-monitor.js

(STABLE) This script is meant to be launched by the UnitTest extension of Wakanda Studio when running in test mode. It instantiates a WebSocket server that acts as a relay between the UnitTest extension and the module:BuildRunner module. It allows the test agent to send commands to the Studio and receive its events in real-time.
