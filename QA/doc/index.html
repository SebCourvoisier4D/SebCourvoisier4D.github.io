Wakanda QA Internal Tools

Cf. https://project.wakanda.org/issues/16676

Purpose

The aim is to develop a single package of node.js modules that provides all the tools needed by the QA team to run the tests and report their results (either programmatically or using the command line):

  • Config (qa-config): handle the loading of the local configuration file used by most of the other components
  • Synchronizer (qa-synchronizer): handle the sync of tests and other assets across the test machines (through Perforce, GitHub, etc.)
  • BuildRetriever (qa-build-retriever): handle the checking of TeamCity status, the retrieval and dispatch of the new builds as soon as they're made available
  • LicenseUpdater (qa-license-updater): handle the update of 4D licenses (and maybe Wakanda licenses one day)
  • BuildRunner (qa-build-runner): handle the running of the required Wakanda and/or 4D builds (prior to the launch of the tests)
  • TestRunner (qa-test-runner): handle the running of the required tests (could be a wrapper around Karma)
  • TestReporter (qa-test-reporter): handle the reporting of the test results to the Dashboard
  • CrashReporter (qa-crash-reporter): handle the reporting of crashs during tests
  • Scripts (qa-scripts): utility scripts (not modules). The new Wakbot will be one of them (and use the above modules)
  • etc.

The goal is to be able to build the new Wakbot like this:

var qa = require('QA'),
    config = new qa.Config(),
    synchronizer = new qa.Synchronizer(),
    buildRetriever = new qa.BuildRetriever(),
    licenseUpdater = new qa.LicenseUpdater();

config.on('config loaded', function (...) {
    synchronizer.sync(...);
});

synchronizer.on('sync done', function (...) {
    buildRetriever.start(...);
});

buildRetriever.on('new build', function (...) {
    licenseUpdater.update(...);
});

licenseUpdater.on('license updated', function (...) {
    // etc.
});

config.load();
